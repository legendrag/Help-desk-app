# دليل النشر والتشغيل لنظام DeskPlus (Monolithic Guide)

هذا الدليل يشرح كيفية تشغيل النظام في بيئات مختلفة بعد تحويله إلى هيكلية Django الموحدة.

---

## 1. المتطلبات الأساسية (Prerequisites)

| المتطلب | الإصدار الأدنى | ملاحظات |
|---|---|---|
| **نظام التشغيل** | Windows 10/11 | 64-bit |
| **Python** | 3.12+ | يجب أن يكون في الـ PATH |
| **MySQL Server** | 8.0+ | أو استخدام XAMPP |

---

## 2. إعداد قاعدة البيانات (Production MySQL)

قبل تشغيل النظام لأول مرة على السيرفر، يجب إنشاء قاعدة البيانات يدوياً عبر أي أداة (مثل phpMyAdmin أو MySQL Workbench أو Command Line):

### خطوات الإنشاء:
1. افتح نافذة الأوامر الخاصة بـ MySQL (أو Query Tab).
2. نفذ الأمر التالي لإنشاء قاعدة البيانات بالترميز الصحيح:
   ```sql
   CREATE DATABASE deskplus CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
3. **تنبيه:** تأكد من أن المستخدم (Root أو غيره) لديه كافة الصلاحيات على هذه القاعدة.

### الربط في النظام:
بعد التثبيت، افتح ملف `.env` وقم بتحديث البيانات التالية:
```env
DB_ENGINE=mysql
DB_NAME=deskplus
DB_USER=root
DB_PASSWORD=كلمة_مرور_قاعدة_البيانات
DB_HOST=localhost
DB_PORT=3306
```

---

## 3. التشغيل المحلي (Development)

1. قم بإنشاء بيئة افتراضية: `python -m venv .venv`
2. تفعيل البيئة: `.\.venv\Scripts\Activate.ps1`
3. تثبيت المكتبات: `pip install -r requirements.txt`
4. تشغيل السيرفر: `python manage.py runserver`

أو استخدم السكريبت الجاهز:
```powershell
.\test-local.ps1 -InstallDeps
```

---

## 4. بناء ملف التثبيت (Windows Installer) 🛠️

إذا كنت تريد توزيع النظام كملف تثبيت (`.exe`) واحد للعملاء:

### المتطلبات الإضافية للبناء:
- تثبيت برنامج **Inno Setup 6** على جهاز المطور.

### خطوات البناء:
1. افتح PowerShell في مجلد المشروع الرئيسي.
2. شغل سكريبت البناء:
   ```powershell
   .\installer\build.ps1
   ```
3. سيقوم السكريبت بجمع الملفات (collectstatic) وإنشاء ملف التثبيت.
4. ستجد الملف الناتج في: `installer\Output\DeskPlusSetup.exe`.

---

## 5. النشر عند العميل (Client Deployment) 🚀

بما أن النظام أصبح Monolith، فإن النشر بسيط جداً:

1. **التثبيت:** انقل ملف `DeskPlusSetup.exe` وكبسه مرتين للتثبيت.
2. **إعدادات البيئة:** افتح ملف `.env` في مجلد التثبيت واضبط البيانات (DB_PASSWORD, SECRET_KEY, SMTP).
3. **التشغيل:** استخدم أيقونة الاختصار "DeskPlus" على سطح المكتب.
4. **التشغيل اليدوي (للمحترفين):**
   يمكنك استخدام `Daphne` لتشغيل النظام لدعم الـ WebSockets:
   ```powershell
   daphne -p 8000 config.asgi:application
   ```

### لماذا نستخدم Daphne؟ 🤔
في وضع الإنتاج (Production)، نستخدم **Daphne** بدلاً من السيرفر العادي لعدة أسباب:
- **دعم الشات اللحظي:** هو الوحيد الذي يفهم تقنية الـ WebSockets المستخدمة في المحادثات والإشعارات.
- **القوة:** مصمم للتعامل مع عدد كبير من الطلبات المتزامنة بكفاءة عالية.
- **التشغيل الموحد:** يقوم بتشغيل واجهة الموقع ومنطق التذاكر وقنوات التواصل في وقت واحد.

---

لفتح البرنامج من أجهزة أخرى في نفس الشبكة:
1. احصل على IP السيرفر (مثلاً `192.168.1.10`).
2. افتح المنفذ 8000 في جدار الحماية (Firewall).
3. الرابط سيكون: `http://192.168.1.10:8000`

---

## 7. ملاحظات هامة
- **البريد الإلكتروني:** تأكد من ضبط إعدادات SMTP في `.env` لتلقي الإشعارات.
- **الأمان:** لا تنسَ تغيير `SECRET_KEY` إلى قيمة معقدة قبل النشر.
